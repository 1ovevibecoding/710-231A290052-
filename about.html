1 Bài tập 1 — Danh sách sản phẩm & Tìm kiếm

Cách xử lý input người dùng để tránh lỗi injection

  • XSS (Cross-Site Scripting): Khi render sản phẩm ra DOM, toàn bộ nội dung người dùng nhập được xử lý qua hàm escapeHtml() — thay thế các ký tự đặc biệt (<, >, &, ", ') thành HTML entities tương ứng. Điều này ngăn chặn attacker chèn thẻ <script> vào ô tìm kiếm.
  • Sanitize input: Hàm sanitizeInput() sử dụng document.createElement('div') + textContent để loại bỏ mọi HTML tag trước khi xử lý tìm kiếm. Kết hợp .trim() để xóa khoảng trắng thừa.
  • Không dùng innerHTML trực tiếp với user input: Dữ liệu sản phẩm được escape trước khi đưa vào innerHTML. Trong thực tế nên dùng textContent hoặc thư viện template engine (như Handlebars) có auto-escaping.

Tối ưu tìm kiếm

  • Case-insensitive: Dùng .toLowerCase() trên cả input và tên sản phẩm để tìm kiếm không phân biệt hoa thường.
  • Debounce: Sử dụng setTimeout với 200ms delay để tránh gọi search liên tục mỗi keystroke — giảm tải render.
  • Filter trên mảng: Dùng Array.prototype.filter() + String.prototype.includes() — độ phức tạp O(n) với n là số sản phẩm. Với dataset nhỏ (< 1000 items) rất hiệu quả.

Code minh họa

function sanitizeInput(str) {
  const div = document.createElement('div');
  div.textContent = str;
  return div.textContent.trim().toLowerCase();
}

// Debounce search
let timeout;
searchInput.addEventListener('input', () => {
  clearTimeout(timeout);
  timeout = setTimeout(searchProducts, 200);
});

2 Bài tập 2 — Form Đăng ký & Bảo mật

Cách xử lý sự kiện submit

  • preventDefault(): Gọi e.preventDefault() trên sự kiện submit để ngăn form reload trang — cho phép validate hoàn toàn bằng JavaScript phía client.
  • Validate tuần tự: Kiểm tra từng trường theo thứ tự: tên → email → mật khẩu → checkbox. Dừng ngay ở lỗi đầu tiên bằng return showError(), tránh hiển thị quá nhiều thông báo lỗi cùng lúc.
  • Email validation: Dùng regex /^[^\s@]+@[^\s@]+\.[^\s@]+$/ kiểm tra định dạng email cơ bản (có @, có domain). Trong thực tế nên bổ sung server-side validation.
  • Password validation: Regex /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/ đảm bảo mật khẩu ≥ 8 ký tự, có ít nhất 1 chữ hoa, 1 chữ thường, 1 số, sử dụng lookahead assertions.

Bảo mật dữ liệu cục bộ (LocalStorage)

  • Không lưu mật khẩu: Trong code, chỉ lưu name, email, registeredAt vào LocalStorage. Mật khẩu KHÔNG được lưu dưới dạng plaintext — trong thực tế, mật khẩu phải được hash (ví dụ: bcrypt) ở phía server.
  • Hạn chế của LocalStorage: Dữ liệu lưu trong LocalStorage không được mã hóa, bất kỳ JavaScript nào chạy trên cùng origin đều có thể đọc được. Do đó, không nên lưu thông tin nhạy cảm (token, password) trong LocalStorage cho production app.
  • try/catch: Wrap LocalStorage operations trong try/catch để xử lý trường hợp storage đầy hoặc bị disabled (private browsing mode).

Code minh họa

// Chỉ lưu info không nhạy cảm
const userData = {
  name: name,
  email: email,
  registeredAt: new Date().toISOString()
};

try {
  const users = JSON.parse(localStorage.getItem('registered_users') || '[]');
  users.push(userData);
  localStorage.setItem('registered_users', JSON.stringify(users));
} catch (e) {
  console.error('LocalStorage error:', e);
}

3 Bài tập 3 — Đồng hồ Đếm ngược & Memory Leak

Xử lý interval để tránh memory leak

  • clearInterval trước khi setInterval mới: Mỗi lần gọi startTimer(), luôn clearInterval(timerInterval) trước khi tạo interval mới. Nếu không, mỗi lần bấm Start sẽ tạo thêm 1 interval chồng chất → timer chạy nhanh gấp đôi, gấp ba.
  • Cleanup khi hết giờ: Khi totalSeconds <= 0, gọi clearInterval() và đặt timerInterval = null ngay lập tức — giải phóng reference.
  • Biến flag isPaused: Sử dụng biến boolean isPaused để kiểm tra trạng thái, tránh tạo nhiều interval khi user bấm Start nhiều lần.

Tích hợp với Domain/Hosting nếu deploy

  • Đồng bộ thời gian server: Trong thực tế, countdown nên lưu thời điểm kết thúc (end time) trên server (API) thay vì đếm local. Client sẽ tính remaining = endTime - Date.now(), đảm bảo nhiều user cùng thấy chung thời gian.
  • Xử lý tab không hoạt động: Khi tab bị minimize, browser throttle setInterval. Giải pháp: dùng Date.now() để tính thời gian thực tế thay vì tin vào mỗi lần gọi callback.
  • Service Worker / Server-Sent Events: Cho ứng dụng đấu giá, thi trắc nghiệm, có thể dùng WebSocket hoặc SSE để server push thời gian còn lại về client, đảm bảo đồng bộ chính xác.

Code minh họa

function startTimer() {
  if (!isPaused) return; // Ngăn tạo nhiều interval
  isPaused = false;

  // Clear interval cũ → tránh memory leak
  if (timerInterval) clearInterval(timerInterval);

  timerInterval = setInterval(() => {
    if (totalSeconds <= 0) {
      clearInterval(timerInterval);
      timerInterval = null; // Giải phóng reference
      isPaused = true;
      showModal('timer-modal');
      return;
    }
    totalSeconds--;
    updateTimerDisplay();
  }, 1000);
}
